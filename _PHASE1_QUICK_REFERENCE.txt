WEEKLY GATES:

THE 5 RULES (NON-NEGOTIABLE):
  1. Readiness Audit MUST pass 100% before canary
  2. ZERO unexplained divergences blocks progression
  3. Fail-Safe mode ON at 10% canary (typed fails → passthrough)
  4. Load test at peak + buffer, NOT average
  5. Post-Week 5: ANY passthrough call = page on-call

THE 4 GAPS (EXPLICITLY DEFINED):
  1. Semantic Match: Allowed (ordering, ±5s timestamps) vs Blocked (missing fields)
  2. Schema Freeze: No changes Weeks 3-5 (critical bugs only, with sign-off)
  3. Golden Dataset: Capture 20-50 scenarios before Week 3 staging
  4. Policy Impact: Track denials/limits separately (not output divergence)

WHAT COUNTS AS MATCH:
  ✅ Exact same output
  ✅ Ordering difference (array reordered)
  ✅ Timestamp ±5 seconds
  ✅ Optional extra fields (typed superset)
  ✅ Type coercion (180 vs 180.0)
  ✅ Policy side-effects (quota exceeded, rate limited, unauthorized)

WHAT COUNTS AS DIVERGENCE (BLOCKER):
  ❌ Missing fields
  ❌ Different values
  ❌ Changed status
  ❌ Structure wrapper change
  ❌ Success vs failure

IF SOMETHING GOES WRONG:

Divergence detected:
  1. Check _PHASE1_SEMANTIC_MATCH_SPEC.md for allowed differences
  2. If allowed → update SEMANTIC_MATCH in DivergenceDetector
  3. If blocked → investigate and fix implementation
  4. DO NOT proceed to next stage if divergence unresolved

Failsafe triggers at 10%:
  1. Check error logs for why typed is failing
  2. Fix implementation
  3. Reset failsafe counter
  4. Re-run load test
  5. Restart from 10%

Load test fails:
  1. Identify at what concurrency p99 exceeds 2000ms
  2. That concurrency is UNSAFE
  3. Improve typed implementation
  4. Re-run load test
  5. Must show safe_ceiling >= peak + buffer
