THREE-LAYER MODEL - ENFORCEMENT RULES
================================================================

LAYER 1: execution_path = "MCP" (ENFORCED)
  Rule: Only value accepted is "MCP"
  Violation: Any non-MCP execution_path → REJECT
  Reason: Single unified authority principle

LAYER 2: transport ∈ {BRIDGE, CLI, MCP_INTERNAL} (ENFORCED)
  Rule: Must be one of defined transport types
  Violation: Unknown transport → REJECT
  Reason: Traceability and origin tracking

LAYER 3: entry_point ≠ "unspecified" (ENFORCED)
  Rule: Must declare specific operation intent
  Violation: Unspecified entry_point → REJECT
  Reason: Intent-based access control foundation

STATE MUTATION ENFORCEMENT (ENFORCEMENT_READY=TRUE):
  Rule: State mutations only from MCP (execution_path="MCP")
  Violation: State mutation from non-MCP → BLOCK
  Reason: Prevent unauthorized data changes
  Example Block: "State mutation blocked: execution_path=CLI is not MCP 
                 authority. Mutations must route through MCP."

================================================================
DRIFT DETECTION - FOUR CATEGORIES
================================================================

1. SCHEMA DRIFT
   Detects: Missing fields, extra fields, type changes
   Threshold: Any mismatch → FLAG
   Example: Real has {id, name, timestamp}, Shadow has {id, name}
   
2. BEHAVIORAL DRIFT  
   Detects: Same input → different output, success/failure mismatch
   Threshold: Any mismatch → FLAG
   Example: Real returns {status: "ok", value: 100}, 
            Shadow returns {status: "ok", value: 200}
   
3. PARTIAL TRUTH (Silent Corruption)
   Detects: Truncated data (>30% null fields, "..." markers)
   Threshold: >30% nulls or truncation markers → FLAG
   Example: {field1: null, field2: null, field3: "truncated..."}
   
4. LATENCY PATHOLOGY
   Detects: P99+ tail latency divergence, burst clustering
   Threshold: |real_latency - shadow_latency| > 1000ms → FLAG
   Example: Real=100ms, Shadow=5000ms (4900ms divergence)
